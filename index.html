How it works

1

Something touches your machine

A honeypot catches an SSH login attempt, an HTTP scan, or a port probe. Or you feed it a log file.

2

Retro looks it up

It asks public sources: who owns that IP? Is it known for abuse? What else is linked to it?

3

A profile is built

Repeat attackers get a file. Retro remembers them, tracks their behaviour, and scores their threat level.

4

You decide what happens

Block the IP, report it, export a threat feed, generate a report. Or let Retro do it automatically.

Frequently asked questions

Does it need internet?

Sensors work fully offline. Some OSINT features (Shodan, VirusTotal) need internet, but DNS and WHOIS work without it.

Is it really free?

Yes. No subscriptions, no cloud credits, no hidden costs. Just Python and optionally free API keys for some sources.

Do I need API keys?

Only for Shodan, VirusTotal, and AbuseIPDB — all have generous free tiers. The core features work with zero keys.

Is this legal?

Yes. Honeypots capture data from people connecting to your machine. Countermeasures act on your own firewall. OSINT uses public data.

What OS does it support?

Linux is recommended for firewall features. The Python backend runs on any OS. The dashboard is a web interface.
